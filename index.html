🔌 客户端接入总览

兼容两种主流格式，切换 Base URL 即可

Base URL

格式Base URL适用场景
OpenAI https://patchtick.com/v1 ChatGPT 类应用、大多数开源工具
Anthropic https://patchtick.com/anthropic Claude 原生 SDK、需要 Anthropic 格式的工具

快速验证（cURL）

# OpenAI 格式 curl https://patchtick.com/v1/chat/completions \ -H "Authorization: Bearer sk-你的Key" \ -H "Content-Type: application/json" \ -d '{ "model": "gpt-4o", "messages": [{"role":"user","content":"Hi"}] }'
# Anthropic 格式 curl https://patchtick.com/anthropic/v1/messages \ -H "x-api-key: sk-你的Key" \ -H "anthropic-version: 2023-06-01" \ -H "Content-Type: application/json" \ -d '{ "model": "claude-sonnet-4-20250514", "max_tokens": 1024, "messages": [{"role":"user","content":"Hi"}] }'

📱 可接入应用 & 极简教程

所有支持自定义 API 地址的应用均可接入

ChatBox

设置 → AI 供应商 → OpenAI API → Base URL 填 https://patchtick.com/v1 → API Key 填你的 Key → 保存

NextChat (ChatGPT Next Web)

设置 → 接口地址填 https://patchtick.com → API Key 填入 → 自定义模型名填需要的模型

Lobe Chat

设置 → 语言模型 → OpenAI → 代理地址填 https://patchtick.com/v1 → 填入 Key → 选模型

Open WebUI

管理面板 → 设置 → 外部连接 → OpenAI API URL 填 https://patchtick.com/v1 → 填 Key → 保存

Cursor / Windsurf (IDE)

Settings → Models → OpenAI API Key 填入 → Override Base URL 填 https://patchtick.com/v1 → 启用模型

Claude 官方 SDK (Python)

client = Anthropic(api_key="你的Key", base_url="https://patchtick.com/anthropic")

OpenAI 官方 SDK (Python)

client = OpenAI(api_key="你的Key", base_url="https://patchtick.com/v1")

沉浸式翻译

设置 → 翻译服务 → OpenAI → 自定义 API 地址填 https://patchtick.com/v1/chat/completions → 填 Key

Bob (macOS)

偏好设置 → 服务 → 添加 OpenAI 翻译 → API URL 填 https://patchtick.com → 填 Key → 选模型

任何兼容 OpenAI 的应用

找到「API Base URL」或「Endpoint」设置项 → 替换为 https://patchtick.com/v1 → 填入你的 Key 即可

🚨 错误码参考

遇到错误时对照此表排查

错误码 含义 处理方式
400 请求格式错误 检查 JSON 格式、必填字段（model, messages）是否正确
401 认证失败 检查 Key 是否正确、Header 格式是否为 Bearer sk-xxx
402 余额不足 前往控制台充值后重试
403 无权访问该模型 确认套餐是否包含该模型，或联系客服开通
404 模型不存在 检查 model 字段拼写，参考 GET /v1/models 获取可用列表
429 请求频率超限 降低请求频率，或升级套餐提高 RPM 上限
500 服务器内部错误 稍后重试；若持续出现请联系客服并提供 request-id
502 上游服务不可用 上游模型暂时故障，等待 1-2 分钟后重试
503 服务维护中 关注公告，通常几分钟内恢复
💡 提示：所有错误响应的 body 中包含 "error": {"message": "...", "code": "..."} 字段，可直接读取具体原因。
